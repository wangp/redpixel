1.2 System Requirements

  • two Pentium class machines
  • DOS or Linux (Intel only, I think)
  • serial cable
  • VGA-compliant card and monitor
  • 8MB RAM
  • mouse
  • tolerance for gore
  • headphones recommended

1.3 What's new in this version

New in 0.11:

  • Linux port and maps by Peter Sundling (kaffepeter@telia.com)
  • stats.fst file
  • three maps by Mike (see maps/authors.txt)
  • mouse wheel support (Linux only)
  • keyboard control in menu
  • demo recording and playback

New in 0.9:

  • a lot more blood and sparks, and better particle generation
  • added scanner powerup
  • added new death sequence (fountain)
  • added cool backdrop for frag count
  • added --lcd option
  • right mouse button now drops mine
  • Q/E and R/Y now select next/prev weapons
  • tweaked stats (see below)
  • configurable stats system
  • minor aesthetic and code-wise fixes
  • four new maps (castle, dome, sniper, mario)
  • changed licence to LGPL
  • this manual

The biggest change version 0.9 (gameplay-wise) is the new stats for items. This was designed to make frags easier, and to make the game more fun. If you don't like the new style, note that you can get (almost) the same stats as in older versions with red -s stats.07. See section Stats for more information.

1.4 Installing

If you are playing under DOS using the binary distribution, you can skip this section.

You will need to firstly install Allegro (WIP 3.9.x or above), and either djgpp or gcc. Then you should be able to simply type ``make'' in the game's root directory.

Under Linux console you will need to run the binary with root permissions. See `readme.lnx' in the Allegro distribution for more information.
